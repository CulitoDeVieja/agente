# Decisión: dashboard-customization

**Decisión:** Sin customización v0.1. Orden de cards fijo: orchestrator, skills-curator, builder, auditor-ops.

## Alternativas
- Drag to reorder — overkill 4 cards.
- Hide/show cards — siempre se quieren ver las 4.
- Layout por usuario — user único.

## Justificación
Orden canónico basado en flujo de agentes (owner-facing → autor de código → QA).
