# Bun vs npm vs pnpm — 2026

## TL;DR
- **Bun**: más rápido (5-17x). 50 deps en 0.8s. Ideal proyectos nuevos sin edge-cases Node.
- **pnpm**: mejor balance. 2-3x más rápido que npm + 50-70% menos disco. Estándar en monorepos.
- **npm**: máxima compatibilidad. Default Node, lento pero seguro para legacy.

## Detalles

**Benchmarks install (warm cache, 2026):**

| Proyecto | bun | pnpm | npm |
|---|---|---|---|
| 50 deps | 0.3s | 1.1s | 3.2s |
| 800 deps | 1.2s | 8.4s | 24.6s |

**Disco:**
- pnpm: content-addressable store → una copia por version global. 50-70% menos que npm.
- Bun: similar a pnpm en ahorro.
- npm: duplica todo por proyecto.

**Monorepos:**
- pnpm workspaces = estándar de facto (Turborepo, Nx lo recomiendan).
- Bun: hoisting estilo npm, funciona si ya funcionaba con npm.
- npm workspaces: ok pero lento con muchos packages.

**Recomendación por caso:**
- **Solo speed, greenfield:** Bun.
- **Balance / monorepo / disco:** pnpm.
- **Compat total Node 24+:** npm.

## Aplicabilidad a panel-agentes
**Sí — considerar migración a pnpm** en v0.2. El panel está en npm por default de `create-tauri-app`. Pasar a pnpm:
1. Disco: VPSs con SSD chico se benefician.
2. Speed: rebuilds en CI más rápidos.
3. Monorepo-ready si en el futuro mergeamos `agente` + `panel-agentes` + otros en un monorepo.

**No migrar a Bun ahora:** Tauri + Rust interacciona con Node toolchain para el build. Bun tiene edge-cases con plugins Babel/Vite específicos. Riesgo > beneficio para v0.1/v0.2.
